## 1. 接口规范

### 1.1 协议说明
- **协议**: HTTP/HTTPS
- **数据格式**: JSON
- **编码**: UTF-8
- **请求方法**: POST（统一使用POST）
- **超时时间**: 10s

### 1.2 响应格式

**成功响应**:
```json
{
  "code": 0,
  "data": {},
  "message": "success"
}
```

**失败响应**:
```json
{
  "code": 1,
  "data": null,
  "message": "错误信息"
}
```

### 1.3 状态码说明

| code | 说明 |
|------|------|
| 0 | 成功 |
| 1 | 通用错误 |
| 1001 | 参数错误 |
| 1002 | 数据不存在 |
| 1003 | 数据已存在 |
| 1004 | 操作失败 |
| 2001 | 未授权 |

## 4. 错误处理

### 4.1 通用错误处理
- 所有接口统一返回 `{code, data, message}` 格式
- 前端根据 code 判断成功/失败
- 根据 message 显示错误提示

### 4.2 常见错误码

| 错误码 | 错误信息 | 处理建议 |
|--------|---------|---------|
| 1001 | 参数错误 | 检查请求参数格式 |
| 1002 | 数据不存在 | 提示用户数据不存在 |
| 1003 | 数据已存在 | 提示用户数据已存在 |
| 1004 | 操作失败 | 提示用户操作失败，请重试 |
| 2001 | 未授权 | 提示用户重新授权 |

## 5. 数据同步

### 5.1 同步策略
- **实时同步**: 打卡、添加习惯等操作立即同步到云数据库
- **本地缓存**: 孩子信息、习惯列表等数据本地缓存
- **离线支持**: 离线时记录操作，联网后自动同步

### 5.2 冲突处理
- 服务器时间优先
- 最后操作覆盖原则
- 关键操作（打卡）不可重复

## 6. 安全性

### 6.1 身份验证
- 基于微信 OpenID 识别用户
- 云函数中通过 `cloud.getWXContext().OPENID` 获取

### 6.2 数据隔离
- 家庭数据隔离（familyId）
- 孩子数据隔离（childId）
- 用户只能访问自己家庭的数据

### 6.3 数据加密
- 敏感数据传输加密（HTTPS）
- 数据库安全配置
