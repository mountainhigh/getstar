# 数据库权限配置指南

## 问题描述

当前 `children` 集合使用默认权限（仅创建者可读写），这会导致：
- 用户 A 创建的孩子，用户 B 无法查看
- 即使使用 `familyId` 查询，也会被 `_openid` 权限限制

## 解决方案

### 方法 1: 修改 children 集合权限（推荐）

访问 CloudBase 控制台：
https://tcb.cloud.tencent.com/dev?envId=cloud1-4g6msqhd5bbad708#/db/doc

1. 找到 `children` 集合
2. 点击"权限设置"
3. 选择"自定义安全规则"
4. 修改为以下规则：

```json
{
  "read": "true",
  "write": "auth != null"
}
```

**说明**：
- `read`: `true` - 允许任何人读取（通过 familyId 控制业务权限）
- `write`: `auth != null` - 仅登录用户可写入

### 方法 2: 使用云函数查询（更安全）

如果担心数据安全，可以创建一个云函数来查询：

```javascript
// cloudfunctions/getChildrenByFamily/index.js
const cloud = require('wx-server-sdk');
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV });
const db = cloud.database();

exports.main = async (event, context) => {
  const { familyId } = event;
  const wxContext = cloud.getWXContext();
  const OPENID = wxContext.OPENID;

  // 验证用户是否属于该家庭
  const memberRes = await db.collection('family_members').where({
    familyId: familyId,
    userId: db.command.in([
      db.collection('users').where({ openid: OPENID }).get().data[0]._id
    ]),
    status: 'active'
  }).get();

  if (memberRes.data.length === 0) {
    return {
      success: false,
      message: '无权访问该家庭数据'
    };
  }

  // 查询孩子列表
  const childrenRes = await db.collection('children').where({
    familyId: familyId,
    isDeleted: false
  }).orderBy('createTime', 'asc').get();

  return {
    success: true,
    data: childrenRes.data
  };
};
```

## 需要修改权限的集合

| 集合 | 当前权限 | 建议权限 | 说明 |
|------|---------|---------|------|
| children | 仅创建者 | 自定义规则 | 家庭成员共享 |
| habits | 仅创建者 | 自定义规则 | 家庭成员共享 |
| check_ins | 仅创建者 | 自定义规则 | 家庭成员共享 |
| families | 仅创建者 | 保持不变 | 通过 family_members 控制 |
| family_members | 仅创建者 | 自定义规则 | 需要查询验证关系 |

## 注意事项

1. **业务层权限控制**：数据库权限放宽后，需要在业务代码中通过 `familyId` 和 `family_members` 表进行权限验证

2. **敏感操作**：删除、修改等操作建议通过云函数进行，确保用户有权限

3. **数据隔离**：虽然数据库权限放宽，但通过 `familyId` 查询仍然可以保证数据隔离
