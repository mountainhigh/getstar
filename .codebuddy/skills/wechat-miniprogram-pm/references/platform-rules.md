## 一、账号与主体规范 {#账号}

### 注册类型
| 主体类型 | 适用场景 | 可用功能差异 |
|---------|---------|------------|
| 个人 | 个人开发者 | 无法使用微信支付、不能认证 |
| 企业 | 商业产品 | 全功能可用，需营业执照 |
| 政府/媒体/其他组织 | 相应机构 | 特定能力需申请 |

### 小程序数量限制
- 同一主体：企业最多 **50** 个，个人最多 **5** 个
- 同一开发者账号：最多关联 **10** 个小程序

### 名称规则
- 长度：4~30 个字符
- 禁止：纯数字、与已有小程序/公众号重名、含平台/微信等字样
- 认证主体可使用与品牌一致的名称

## 二、功能与内容规范 {#内容}

### 严禁内容（一票否决）
- 政治敏感内容
- 色情、赌博、诈骗
- 未经授权的版权内容
- 虚假信息、误导性内容
- 恶意代码、病毒

### 服务类目要求
以下类目需要额外资质：
| 类目 | 所需资质 |
|------|---------|
| 医疗健康 | 医疗机构执业许可证 |
| 金融（贷款/保险等） | 相应金融牌照 |
| 教育（培训/考试） | 办学许可证 |
| 游戏 | 版号（仅企业可申请） |
| 直播 | 互联网视听节目服务许可证 |
| 点餐/外卖 | 食品经营许可证 |

### 用户隐私规范
- **必须**有隐私政策，且需在首次使用前展示并获取同意
- 收集用户信息需「最小必要原则」，并说明用途
- 敏感权限（定位、相机、麦克风等）需在使用时动态申请，禁止后台静默获取
- 不得将用户数据出售给第三方

### 支付规范
- 虚拟商品/增值服务**必须**使用微信支付，禁止引导至外部支付
- 实物商品可以使用其他支付方式
- 禁止在小程序内诱导用户绕过平台进行交易

## 三、开放能力使用规范 {#开放能力}

### 登录能力
```
wx.login() → 获取 code → 后端换取 session_key + openid
```
- 禁止将 `session_key` 传到前端
- 禁止将 `openid` 直接作为用户唯一标识用于敏感操作（需配合登录态）

### 获取用户信息
- **微信头像/昵称**：需用户主动点击 `<button open-type="chooseAvatar">` 授权
- 禁止静默获取用户真实头像和昵称（旧接口已废弃）
- UnionID 需要绑定开放平台账号才能获取

### 订阅消息
- 用户点击允许后，每次订阅只能发送 **1 条**
- 长期订阅需用户每次手动授权
- 消息内容必须与订阅场景一致，禁止营销推广

### 客服消息
- 用户主动发消息后，**48 小时内**可以回复
- 可发送文字、图片、小程序卡片等
- 禁止发送骚扰、广告等内容

### 位置能力
- 使用前必须申请 `scope.userLocation`
- 在 `app.json` 中声明 `permission.scope.userLocation.desc`
- 后台定位需额外申请 `scope.userLocationBackground`

### 分享能力
- 页面分享：`onShareAppMessage` / `onShareTimeline`（需在后台开通朋友圈分享）
- 禁止使用诱导分享语言（「分享得奖励」须符合运营规范）
- 分享卡片标题不得含诱导性词语

## 四、审核常见拒绝原因 {#审核}

### 功能类
| 拒绝原因 | 处理方案 |
|---------|---------|
| 功能不完整，存在无法使用的页面 | 提交前完整测试所有页面 |
| 类目与实际功能不符 | 重新选择准确类目，补充资质 |
| 涉及需要资质的功能但未上传证件 | 在后台上传对应资质文件 |
| 支付流程不规范 | 虚拟商品必须走微信支付 |

### 内容类
| 拒绝原因 | 处理方案 |
|---------|---------|
| 没有隐私政策 | 添加隐私政策页面，首次打开弹窗提示 |
| 用户协议缺失 | 注册/登录页面添加用户协议链接 |
| 含有外链跳转 | 移除或替换 web-view 中不合规的外部链接 |
| 诱导分享/关注 | 修改相关文案，去掉强制引导 |

### 技术类
| 拒绝原因 | 处理方案 |
|---------|---------|
| 调用未声明的权限 | 在 `app.json` 的 `permission` 字段中声明 |
| 使用了废弃接口 | 升级到最新 API |
| 体验版无法正常访问 | 检查服务端是否允许审核 IP 访问 |

## 五、版本管理规范 {#版本}

### 版本状态流转
```
开发版 → 体验版 → 审核版 → 线上版
```

| 状态 | 说明 | 访问权限 |
|------|------|---------|
| 开发版 | 开发工具上传后 | 开发者 + 体验者 |
| 体验版 | 设置为体验版 | 体验成员（最多15人） |
| 审核版 | 提交审核 | 无法访问 |
| 线上版 | 审核通过并发布 | 所有用户 |

### 版本号规范（语义化）
```
主版本.次版本.修订号
1.0.0 → 初始上线
1.1.0 → 新功能迭代
1.1.1 → Bug 修复
2.0.0 → 重大重构
```

### 灰度发布
- 支持按比例灰度：1% / 5% / 10% / 25% / 50% / 100%
- 灰度期间可监控错误率，异常时可快速回滚
- 建议：新功能 5% 灰度观察 1-2 天后再全量

### 紧急修复流程
1. 开发版修复并充分测试
2. 提交加急审核（说明紧急原因）
3. 通过后立即全量发布
4. 事后补充回归测试

## 六、小程序码与跳转规则 {#跳转}

### 小程序码类型
| 类型 | 特点 | 适用场景 |
|------|------|---------|
| 小程序码 | 圆形，美观，数量无限制 | 线下推广、海报 |
| 小程序二维码 | 方形，传统格式 | 兼容性场景 |
| 短链 | 文字链接，可定制 | 线上分享 |

### 跳转规则
- **小程序间跳转**：`wx.navigateToMiniProgram`，需在后台配置关联小程序
- **H5 跳转小程序**：通过开放标签 `<wx-open-launch-weapp>` 实现
- **App 跳转小程序**：微信开放平台 SDK，需同主体或关联主体
- **小程序跳转 H5**：使用 `web-view` 组件，域名需在后台白名单中

### 场景值（Scene Value）
常用场景值，用于分析用户来源：
| 场景值 | 来源 |
|--------|------|
| 1011 | 扫描小程序码 |
| 1037 | 小程序内跳转 |
| 1044 | 朋友圈广告 |
| 1047 | 扫描小程序二维码 |
| 1089 | 微信聊天主界面下拉 |
| 1090 | 长按图片识别二维码 |
