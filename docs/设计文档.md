## 1. 设计原则

### 1.1 游戏化设计原则
- **即时反馈**：打卡后立即显示积分增加动画和音效
- **进度可视化**：积分、等级进度条清晰展示
- **成就感**：升级、称号解锁时有庆祝动画
- **社交激励**：排行榜激发竞争意识

### 1.2 操作便捷性原则
- **首页直达**：核心功能（打卡）首页直接操作
- **2-3步完成**：点击习惯 → 选择完成 → 确认，最多3步
- **预设模板**：点击模板即可完成习惯添加
- **减少输入**：文字备注为可选，大部分情况一键完成

### 1.3 视觉设计原则
- **明亮活泼**：使用温暖、充满活力的色彩
- **吸引付费**：精致视觉，达到付费产品标准
- **童趣风格**：符合儿童审美，但也让家长觉得专业
- **层次清晰**：信息结构清晰，重点突出

## 2. 产品界面设计

### 2.1 整体架构

```
小程序
├── TabBar
│   ├── 打卡（首页 - 今日打卡）
│   ├── 习惯（习惯管理）
│   ├── 记录（打卡历史）
│   └── 我的（设置、孩子管理、礼物兑换等）
```

**说明**：
- TabBar 共4个页面，符合微信小程序最多5个tab的限制
- 统计功能整合在"记录"页面中
- 礼物兑换功能整合在"我的"页面中（独立页面 `pages/reward-exchange/reward-exchange`）

```
│  │                   │  │

│  └───────────────────┘  │
├─────────────────────────┤
```

**交互说明**:
- 点击"积分"数字 → 弹出数字键盘修改积分（默认显示习惯设置值，可上下调整）
- 点击照片上传区域：
  - 第一次点击 → 直接打开相机（默认拍照模式）
  - 已有照片点击 → 打开相机选择器（可选择：拍照 / 从相册选择）
  - 长按照片 → 删除照片
  - 最多支持上传9张照片
- 点击"完成"按钮 → 上传照片到云存储 → 弹出庆祝动画 → 返回首页
- 动画效果：积分数字滚动增加 + 等级进度条动画 + 音效
- 支持输入备注（可选），默认聚焦到备注框

**技术实现**:
- 使用 `wx.chooseMedia` API 实现拍照/选择照片
- 参数设置：`mediaType: ['image']`, `sourceType: ['camera', 'album']`, `maxDuration: 30`
- 默认优先相机：第一次调用时 `sourceType: ['camera']`
- 上传到 CloudBase 云存储
- 照片压缩：使用 `compress: true` 自动压缩

## 3. 核心流程设计

### 3.1 打卡流程（核心流程）

```
用户操作流程：
1. 打开小程序 → 进入首页
2. 首页显示当前孩子 + 今日待打卡习惯列表
3. 点击习惯卡片 → 进入打卡详情页
4. （可选）点击积分数字修改积分
5. （可选）点击拍照区域上传照片证据
6. （可选）输入备注
7. 点击"完成"按钮 → 上传照片 → 弹出庆祝动画 → 返回首页
8. 首页更新：积分增加、习惯标记为已完成

耗时：< 10秒完成一次打卡（含拍照）
```

### 3.2 添加习惯流程

```
用户操作流程：
1. 首页点击"添加习惯" → 弹出模板选择器
2. 点击某个模板 → 自动添加到当前孩子
3. 返回首页，新习惯显示在列表中

耗时：< 10秒添加一个习惯
```

### 3.3 多孩切换流程

``` 
用户操作流程：
1. 点击首页顶部孩子头像/名称
2. 弹出孩子选择器
3. 点击要切换的孩子
4. 页面数据更新为选中孩子的数据

耗时：< 3秒完成切换
```

### 3.4 礼物兑换流程

```
用户操作流程：
1. 点击TabBar"礼物兑换"
2. 查看可兑换礼物列表
3. 点击某个礼物的"兑换"按钮
4. 弹出确认对话框（显示礼物名称、金币数量）
5. 确认兑换 → 扣除金币 → 返回礼物兑换页面
6. 兑换记录生成，状态为"进行中"
7. 家长线下完成礼物交付
8. 家长在兑换记录中标记为"已完成"

耗时：< 15秒完成一次兑换
```
