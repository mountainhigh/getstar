# UI/UX 设计原则与理论参考

## 一、核心设计原则

### 1. 尼尔森十大可用性启发原则

| # | 原则 | 说明 | 常见违反示例 | 修正方法 |
|---|------|------|------------|--------|
| 1 | 系统状态可见性 | 用户应始终了解当前发生的事情 | 上传文件无进度反馈 | 进度条、状态标签、Toast |
| 2 | 系统与现实世界匹配 | 使用用户熟悉的语言和概念 | 技术术语暴露给普通用户 | 使用日常词汇、隐喻 |
| 3 | 用户控制与自由 | 提供"紧急出口"支持撤销/重做 | 误删无法恢复 | 撤销操作、确认弹窗 |
| 4 | 一致性与标准 | 相同操作应有相同表现 | 部分按钮蓝色部分绿色含义不同 | 建立设计系统 |
| 5 | 防错设计 | 在错误发生前预防 | 表单提交后才报错 | 实时校验、禁用不可用选项 |
| 6 | 识别优于回忆 | 减少用户记忆负担 | 用户必须记住上一步的信息 | 显示上下文、提供提示 |
| 7 | 灵活性与高效性 | 支持新手和专家两种路径 | 只有一种操作方式 | 快捷键、批量操作 |
| 8 | 美观极简设计 | 不展示无关或极少用信息 | 每个页面信息堆砌 | 信息层次化、渐进披露 |
| 9 | 帮助识别和解决错误 | 错误信息应清晰说明问题和解决方案 | "Error 500" | "网络连接失败，请检查网络后重试" |
| 10 | 帮助文档 | 提供易检索的使用说明 | 无任何引导或帮助入口 | FAQ、Tooltip、空状态引导 |

### 2. 格式塔视觉感知原则

- **接近性**：相近元素被视为一组 → 用间距划分功能区
- **相似性**：相似元素被视为一类 → 同类操作用相同样式
- **连续性**：视线倾向沿线条/曲线移动 → 利用对齐引导视线
- **封闭性**：大脑补全不完整的图形 → 图标设计可利用负空间
- **图底关系**：区分前景与背景 → Modal 遮罩、卡片阴影
- **对称性**：对称布局感觉稳定 → 表单、卡片网格

---

### 3. 费茨定律（Fitts's Law）

**核心**: 点击目标的时间与距离成正比，与目标大小成反比。

**实践应用**:
- 重要操作按钮尺寸至少 44×44px（移动端）
- 常用功能放置在用户手指容易触及区域
- 弹出菜单应靠近触发点
- 避免将高频操作放在角落或边缘

---

### 4. 希克定律（Hick's Law）

**核心**: 决策时间随选项数量的增加而增加（对数关系）。

**实践应用**:
- 导航菜单不超过 7±2 个选项
- 分步骤引导复杂操作（分拆决策）
- 默认值减少用户需要做的选择
- 推荐选项高亮降低认知负担

---

### 5. 米勒定律（Miller's Law）

**核心**: 人类短期记忆容量约为 7±2 个组块。

**实践应用**:
- 分组展示信息（信息分块）
- 分页/懒加载限制每页内容量
- 步骤导航不超过 5-7 步

## 二、设计系统基础

### 色彩体系

```
Primary         主色      品牌核心色，用于主要操作
Secondary       辅助色    支撑主色，用于次要操作
Accent          强调色    少量使用，吸引注意力
Neutral         中性色    文本、背景、边框（灰阶）
Semantic        语义色    Success / Warning / Error / Info
```

**对比度标准 (WCAG)**:
- 正文文字 ≥ 4.5:1（AA 级）
- 大号文字 ≥ 3:1（AA 级）
- 增强级正文 ≥ 7:1（AAA 级）

---

### 字体排版规范

**字体比例（Major Third 1.250）**:
```
xs:   12px / 0.75rem
sm:   14px / 0.875rem
base: 16px / 1rem      ← 基准
lg:   20px / 1.25rem
xl:   24px / 1.5rem
2xl:  30px / 1.875rem
3xl:  36px / 2.25rem
4xl:  48px / 3rem
```

**行高规范**:
- 标题：1.2 - 1.3
- 正文：1.5 - 1.6
- 说明文字：1.4

**字重层次**:
- Regular (400)：正文
- Medium (500)：强调正文
- Semibold (600)：小标题
- Bold (700)：标题

---

### 间距系统（8pt Grid）

```
0.5x = 4px
1x   = 8px
1.5x = 12px
2x   = 16px
3x   = 24px
4x   = 32px
5x   = 40px
6x   = 48px
8x   = 64px
10x  = 80px
12x  = 96px
```

**使用原则**:
- 相关元素间距 ≤ 8px（接近性）
- 功能组内间距 12-16px
- 功能组间间距 24-32px
- 页面区块间距 48-64px
- 页面边距 16px（移动）/ 24-32px（平板）/ 40-64px（桌面）

---

### 圆角规范

```
none:  0px      边框/表格
sm:    4px       输入框内部元素
base:  8px       卡片、按钮（默认）
md:    12px      较大卡片
lg:    16px      对话框
xl:    20px      底部弹层
2xl:   24px      大型模态框
full:  9999px    胶囊形标签/按钮
```

|------|---------|---------|
| 蓝色 | 信任、专业、冷静 | 金融、科技、医疗 |
| 绿色 | 健康、成功、自然 | 环保、健康、成功状态 |
| 红色 | 紧迫、危险、热情 | CTA 强调、错误、警告 |

## 四、无障碍设计（Accessibility / a11y）

### WCAG 2.1 四大原则（POUR）
1. **可感知**（Perceivable）：信息能被感知（文字替代图像、字幕、对比度）
2. **可操作**（Operable）：界面可通过键盘操作、有充足时间、无闪烁
3. **可理解**（Understandable）：内容可理解、行为可预测、错误提示可读
4. **健壮性**（Robust）：兼容辅助技术（屏幕阅读器等）

### 实践检查清单
- [ ] 所有图像有 alt 文字
- [ ] 颜色不是唯一信息传递方式
- [ ] 交互元素可通过键盘访问
- [ ] 焦点状态清晰可见（focus ring）
- [ ] 表单输入有关联标签 `<label>`
- [ ] 错误信息明确指出出错位置和修复方法
- [ ] 文字对比度满足 WCAG AA
- [ ] 动效可关闭（prefers-reduced-motion）

## 五、响应式设计断点

### 标准断点
```
xs:   < 480px     手机竖屏
sm:   480-767px   手机横屏
md:   768-1023px  平板
lg:   1024-1279px 小桌面
xl:   1280-1535px 标准桌面
2xl:  ≥ 1536px    大屏
```

### 移动优先原则
1. 先设计移动端（约束优先）
2. 逐步增强到平板、桌面
3. 触控优先：最小触控区域 44×44px
4. 拇指可达区：屏幕底部 2/3 区域优先放置高频操作
