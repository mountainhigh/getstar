## 1. 数据库选型

### 1.1 技术选型
- **数据库**: CloudBase 文档数据库（NoSQL）
- **特点**: 
  - 与微信小程序无缝集成
  - 支持 JSON 文档存储
  - 自动扩展
  - 免费额度充足

## 2. 数据库集合设计

### 2.1 users（用户表）

**集合说明**: 存储用户基本信息

**字段说明**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| _id | String | 是 | 主键（自动生成） |
| _openid | String | 是 | 用户OpenID（系统自动注入） |
| nickname | String | 否 | 用户昵称 |
| avatar | String | 否 | 头像URL |
| familyId | String | 否 | 当前选中的/默认家庭ID（用户可加入多个家庭，此字段记录当前活跃家庭） |
| createTime | Date | 是 | 创建时间 |
| updateTime | Date | 是 | 更新时间 |

---

### 2.2 family_members（家庭成员关系表）

**集合说明**: 存储用户与家庭的多对多关系，支持用户加入多个家庭

**字段说明**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| _id | String | 是 | 主键（自动生成） |
| _openid | String | 是 | 创建者OpenID（系统自动注入） |
| familyId | String | 是 | 家庭ID（关联 families._id） |
| userId | String | 是 | 用户ID（关联 users._id） |
| role | String | 是 | 角色：'owner'（创建者，唯一）/ 'admin'（管理员）/ 'member'（普通成员） |
| nickname | String | 否 | 在该家庭的昵称 |
| joinTime | Date | 是 | 加入时间 |
| status | String | 是 | 状态：'active'（正常）/ 'pending'（待确认）/ 'removed'（已移除） |
| invitedBy | String | 否 | 邀请人ID |
| createTime | Date | 是 | 创建时间 |
| updateTime | Date | 是 | 更新时间 |

**索引建议**:
- `{ familyId: 1, userId: 1 }` 唯一索引：同一用户在同一家庭只能有一条记录
- `{ userId: 1, status: 1 }`：查询用户的所有活跃家庭
