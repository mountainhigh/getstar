## 1. 接口规范

### 1.1 协议说明
- **协议**: HTTP/HTTPS (云函数调用)
- **数据格式**: JSON
- **编码**: UTF-8
- **请求方法**: POST（统一使用POST）
- **超时时间**: 10s

### 1.2 响应格式

**成功响应**:
```json
{
  "code": 0,
  "data": {},
  "message": "success"
}
```

**失败响应**:
```json
{
  "code": 1,
  "data": null,
  "message": "错误信息"
}
```

## 2. 数据交互模式

本项目采用混合模式进行数据交互：

1.  **前端直接数据库操作**: 对于简单的 CRUD 操作（如家庭管理、孩子管理、习惯管理），前端直接使用 `wx.cloud.database()` 操作数据库，利用云开发的权限控制（Security Rules）保证数据安全。
2.  **云函数**: 对于复杂的业务逻辑（如打卡、兑换礼物、排行榜计算）或涉及跨集合事务的操作，使用云函数执行。

### 2.1 前端直接操作的模块
- **家庭管理**: 创建家庭、获取家庭信息
- **孩子管理**: 添加、更新、删除孩子
- **习惯管理**: 添加、更新、删除习惯（不涉及模板导入）
- **统计查询**: 简单的统计数据查询

## 4. 核心云函数详细设计

### 4.1 userLogin - 用户登录

**功能**: 获取当前用户信息，如果不存在则创建用户记录。同时尝试获取关联的家庭信息。

**请求参数**: 无

**响应数据**:
```json
{
  "openid": "用户OpenID",
  "user": { ... }, // 用户信息
  "family": { ... }, // 家庭信息（如果存在）
  "hasFamily": true/false
}
```

### 4.2 checkIn - 打卡

**功能**: 记录打卡行为，增加孩子星星和金币，更新连续打卡天数。

**请求参数**:
```json
{
  "childId": "孩子ID",
  "habitId": "习惯ID",
  "date": "2023-01-01",
  "points": 5,
  "remark": "备注",
  "photos": []
}
```

**逻辑**:
1. 验证习惯是否存在
2. 记录打卡数据到 `check_ins`
3. 更新 `children` 表的 `points`, `coins`, `continuousDays`
4. 记录金币变动到 `coin_records`
5. 异步调用 `checkBadgeUnlock` 检查称号

### 4.3 exchangeReward - 兑换礼物

**功能**: 孩子使用金币兑换礼物。

**请求参数**:
```json
{
  "childId": "孩子ID",
  "rewardId": "礼物ID"
}
```

**逻辑**:
1. 检查孩子金币余额是否充足
2. 检查礼物库存（如果有库存限制）
3. 扣除孩子金币
4. 记录兑换记录到 `reward_exchanges`
5. 记录金币变动到 `coin_records`
6. 减少礼物库存（如果需要）

### 4.4 getLeaderboard - 获取排行榜

**功能**: 计算并返回家庭成员的排行榜。

**请求参数**:
```json
{
  "period": "week" // week, month, total
}
```

**逻辑**:
1. 根据 `period` 计算起始时间
2. 查询家庭内所有孩子
3. 聚合查询 `check_ins` 表，计算每个孩子在时间范围内的总星星
4. 排序并返回
