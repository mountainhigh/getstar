## 模板3：用户流程图描述 {#flow}

```markdown
## 用户流程：[流程名称]

**场景描述**：[用户在什么情境下使用此流程]

### 主流程

```
[入口] 
  ↓
[步骤1：用户操作]
  ↓
[步骤2：系统处理]
  ↓
[步骤3：展示结果]
  ↓
[完成/下一步]
```

### 分支流程

**分支A：[触发条件]**
```
步骤2 → [A条件判断]
  ├── 是 → [处理方式A] → 继续主流程步骤3
  └── 否 → [处理方式B] → 结束/回退
```

**分支B：异常流程**
```
步骤2 → 接口调用失败
  → Toast 提示错误原因
  → 用户可重试
```

### 关键决策点

| 决策点 | 判断条件 | 处理结果 |
|--------|---------|---------|
| 是否登录 | session 是否有效 | 未登录跳转授权页 |
| 是否有权限 | 用户角色判断 | 无权限显示提示 |
| 数据是否完整 | 必填字段校验 | 不完整时阻止提交 |
```

## 模板4：交互说明规范 {#interaction}

```markdown
## 交互说明：[功能/页面名称]

### 动效规范
| 场景 | 动效类型 | 时长 | 缓动曲线 |
|------|---------|------|---------|
| 页面进入 | slide-in-right | 300ms | ease-out |
| 弹窗出现 | fade + scale | 200ms | ease-out |
| 列表加载 | skeleton screen | - | - |
| 按钮点击 | 缩放 0.95 | 100ms | linear |

### 手势交互
- **下拉刷新**：`enablePullDownRefresh: true`，刷新完成后调用 `wx.stopPullDownRefresh()`
- **上拉加载**：`onReachBottom`，距底部 50px 触发，展示 loading 状态
- **左滑操作**：[如删除/编辑，说明 swipe 组件用法]

### 弹窗层级规范
```
遮罩层（z-index: 100）
  ├── 全屏弹窗（z-index: 200）
  ├── 半屏弹窗（z-index: 300）
  ├── Toast（z-index: 400）
  └── Loading（z-index: 500）
```

### 操作反馈规范
| 操作类型 | 反馈方式 | 文案规范 |
|---------|---------|---------|
| 操作成功 | `wx.showToast({ icon: 'success' })` | 动词+名词，如「保存成功」 |
| 操作失败 | `wx.showToast({ icon: 'error' })` | 说明原因，如「网络异常，请重试」 |
| 危险操作 | `wx.showModal` 二次确认 | 「确认删除？此操作不可撤销」 |
| 加载中 | `wx.showLoading` | 「加载中…」或具体操作，如「提交中…」 |

### 空状态设计
每个列表页必须设计空状态，包含：
1. **插图**：与场景相关的空状态图
2. **主文案**：说明为什么为空（「暂无订单」「搜索无结果」）
3. **引导操作**（可选）：按钮引导用户下一步（「去逛逛」「重新搜索」）
```

## 模板5：数据埋点方案 {#tracking}

```markdown
## 数据埋点方案

### 埋点分类
| 类型 | 说明 | 示例 |
|------|------|------|
| 页面 PV/UV | 页面访问量 | home_page_view |
| 功能点击 | 按钮/功能使用 | search_btn_click |
| 业务转化 | 核心业务动作 | order_submit_success |
| 性能监控 | 页面/接口性能 | api_response_time |

### 命名规范
格式：`[模块]_[元素]_[动作]`
- 示例：`home_banner_click`、`order_submit_success`、`profile_setting_view`

### 埋点清单

#### 首页（/pages/home/index）
| 事件名 | 触发时机 | 上报参数 |
|--------|---------|---------|
| `home_page_view` | 页面加载完成 | `{ scene, source }` |
| `home_banner_click` | 点击 Banner | `{ banner_id, position }` |
| `home_search_click` | 点击搜索框 | - |

#### [功能模块]
| 事件名 | 触发时机 | 上报参数 |
|--------|---------|---------|
| ... | ... | ... |

### 核心漏斗定义
```
[入口曝光] → [功能点击] → [关键步骤完成] → [目标转化]

示例（购买漏斗）：
商品详情页_view → 立即购买_click → 订单确认页_view → 支付成功_success
```

### 关键指标定义
| 指标 | 计算公式 | 统计周期 |
|------|---------|---------|
| 次日留存率 | 次日 UV / 首日 UV | 日 |
| 功能渗透率 | 使用该功能 UV / 总 UV | 日/周 |
| 核心转化率 | 目标动作 UV / 入口 UV | 日 |
```

## 模板6：上线 Checklist {#checklist}

```markdown
## 上线 Checklist

### 一、产品自查
- [ ] 所有功能按 PRD 验收通过
- [ ] 空状态、错误状态、加载状态均已设计
- [ ] 文案无错别字，表述清晰
- [ ] 有隐私政策页面，首次打开有弹窗
- [ ] 有用户协议，注册页面可访问
- [ ] 支付流程合规（虚拟商品用微信支付）

### 二、设计验收
- [ ] 视觉还原度符合设计稿
- [ ] 字体/颜色/间距符合规范
- [ ] 图标 / 图片已优化，无模糊
- [ ] 深色模式适配（如有要求）

### 三、技术自查
- [ ] 所有权限在 `app.json` 中声明
- [ ] 隐私接口申请已填写用途说明
- [ ] 使用的第三方 SDK 合规
- [ ] 无废弃接口
- [ ] 服务端接口白名单配置完成（允许审核 IP）
- [ ] HTTPS 域名已在小程序后台配置

### 四、测试验收
- [ ] 主流程冒烟测试通过
- [ ] 边界场景测试通过（空数据、网络断开、权限拒绝）
- [ ] iOS / Android 双端测试通过
- [ ] 微信最新版 / 次新版测试通过
- [ ] 性能测试（首屏加载 < 3s）

### 五、审核准备
- [ ] 体验版功能完整（审核员能操作所有功能）
- [ ] 测试账号准备好（需要登录的功能）
- [ ] 类目选择正确，资质文件已上传
- [ ] 版本说明填写清晰
- [ ] 若有地理位置等权限，说明使用目的

### 六、上线后观察
- [ ] 监控错误率（< 0.1% 为健康）
- [ ] 关注启动性能数据
- [ ] 查看用户反馈/评价
- [ ] 数据埋点数据核对
```
