# KidsStar 需求说明书

**版本**：v1.0  
**日期**：2026-03-01  
**产品名称**：KidsStar 儿童行为激励管理小程序  

---

## 一、项目背景与目标

### 1.1 背景
现代家庭普遍面临孩子习惯培养难题：语言督促效果有限、任务记录缺乏系统性、多孩家庭管理尤为复杂。游戏化机制（星星、等级、称号）已被心理学研究证实能有效提升儿童的内在动机。

### 1.2 产品目标
- 帮助家长以游戏化方式管理孩子日常行为习惯
- 最简化操作路径（2-3步完成核心打卡）
- 支持多孩家庭统一管理
- 打造付费产品级别的视觉吸引力

### 1.3 目标用户
- **主要用户**：3-12 岁儿童的家长（25-40岁）
- **使用场景**：家庭日常、睡前打卡、奖励兑换
- **设备**：iPhone / Android 微信客户端

## 二、功能需求

### 2.1 用户系统

#### REQ-01 微信授权登录
> **User Story**：作为家长，我希望使用微信账号快速登录，无需注册繁琐账号。

**验收标准：**
1. When 用户首次打开小程序，the system shall 展示微信授权登录界面。
2. When 用户点击"微信授权登录"，the system shall 获取微信 openid 并完成注册/登录，整个流程 ≤ 3 秒。
3. While 已登录状态，when 再次打开小程序，the system shall 自动恢复登录态，无需重新授权。
4. When 登录成功，the system shall 跳转至首页并显示用户昵称/头像。

#### REQ-02 家庭信息设置
> **User Story**：作为家长，我希望设置家庭名称，让应用有归属感。

**验收标准：**
1. When 首次登录后，the system shall 引导用户填写家庭名称（可跳过，默认"我的家"）。
2. When 用户保存家庭名称，the system shall 在首页顶部展示家庭名称。
3. The system shall 支持随时在"设置"中修改家庭名称。

---

### 2.2 孩子管理

#### REQ-03 添加孩子
> **User Story**：作为家长，我希望添加多个孩子档案，分别管理他们的行为数据。

**验收标准：**
1. When 用户点击"添加孩子"，the system shall 展示孩子信息填写表单（姓名、昵称、性别、生日、头像）。
2. The system shall 支持最多 **6 个**孩子档案。
3. When 孩子档案保存成功，the system shall 为该孩子自动初始化星星账户（总星星=0，等级=1）。
4. The system shall 为每个孩子提供默认卡通头像（6款可选），也支持从手机相册上传。
5. When 用户删除孩子，the system shall 弹出二次确认弹窗，并提示"历史数据将被保留"。

#### REQ-04 孩子切换
> **User Story**：作为多孩家庭家长，我希望快速切换孩子视角，查看各孩子状态。

**验收标准：**
1. When 首页存在多个孩子，the system shall 在顶部展示孩子头像横向列表，点击即可切换。
2. When 切换孩子，the system shall 在 500ms 内完成页面数据刷新。
3. The system shall 记录上次查看的孩子，下次打开默认激活该孩子。

---

### 2.3 任务与打卡（核心功能）

#### REQ-05 任务模板库
> **User Story**：作为家长，我希望从预设模板中快速选择任务，无需从零配置。

**验收标准：**
1. The system shall 提供 **≥ 20 个**预设任务模板，覆盖以下分类：
   - 学习习惯（按时完成作业、阅读30分钟等）
   - 生活习惯（早起、整理房间、刷牙等）
   - 体育运动（运动30分钟、户外活动等）
   - 品德修养（帮助家人、礼貌待人等）
2. When 用户选择模板，the system shall 自动填充任务名称、默认星星（5-20分）、图标，用户可修改。
3. The system shall 支持创建完全自定义任务（名称、图标、星星、描述）。

#### REQ-06 任务分配与激活
> **User Story**：作为家长，我希望将任务分配给指定孩子，并设置重复周期。

**验收标准：**
1. The system shall 支持任务频率设置：每日、每周指定天、一次性。
2. When 任务频率为"每日"，the system shall 在每天零点自动为该孩子生成当日任务实例。
3. The system shall 支持设置任务有效时间段（如：17:00-21:00 完成有效）。
4. When 任务被停用，the system shall 不再生成新实例，但保留历史记录。

#### REQ-07 打卡申请（孩子视角辅助操作）
> **User Story**：作为家长，我希望家长可以代孩子提交打卡，减少孩子单独操作设备的机会。

**验收标准：**
1. When 家长在首页点击某孩子的任务卡片，the system shall **在 2 步内**完成打卡提交：
   - 第1步：点击任务卡片
   - 第2步：点击"完成打卡"按钮（可附照片，可选）
2. The system shall 支持打卡时上传最多 **1 张**照片作为凭证（可选）。
3. When 打卡提交成功，the system shall 展示动效反馈（星星飞出、星星增加动画）。

#### REQ-08 打卡审核
> **User Story**：作为家长，我希望审核孩子（或自己代提交）的打卡记录，确认后才计分。

**验收标准：**
1. The system shall 支持两种打卡模式（可配置）：
   - **直接计分模式**：打卡即得分，无需审核（默认）
   - **审核模式**：提交后需家长点击"通过"才计分
2. When 处于审核模式且有待审核记录，the system shall 在首页显示红点提醒。
3. When 家长驳回打卡，the system shall 支持填写驳回原因（不超过50字）。
4. The system shall 支持批量审核（全选/一键通过）。

### 2.4 星星系统

#### REQ-09 星星计算
> **User Story**：作为家长，我希望星星规则灵活，能给特殊表现额外加减分。

**验收标准：**
1. When 打卡审核通过（或直接计分），the system shall 自动为该孩子增加对应星星，同时记录星星流水。
2. The system shall 支持**手动加减分**（+1 ~ +100 / -1 ~ -50），需填写原因。
3. The system shall 支持**连续打卡奖励**：连续完成任务 3/7/14/30 天，分别额外奖励 5/15/30/60 分。
4. The system shall 支持**星星有效期**设置（永久 / 按月重置 / 按学期重置），默认永久。

#### REQ-10 等级系统
> **User Story**：作为家长，我希望孩子通过积累星星升级，保持长期动力。

**验收标准：**
1. The system shall 内置 10 个等级，升级所需**累计星星**如下：

| 等级 | 名称 | 所需累计星星 |
|------|------|-------------|
| Lv.1 | 小星星 | 0 |
| Lv.2 | 闪亮星 | 100 |
| Lv.3 | 超级星 | 300 |
| Lv.4 | 金色星 | 600 |
| Lv.5 | 钻石星 | 1000 |
| Lv.6 | 荣耀星 | 1500 |
| Lv.7 | 传奇星 | 2200 |
| Lv.8 | 神话星 | 3000 |
| Lv.9 | 宇宙星 | 4000 |
| Lv.10 | 至尊星 | 5500 |

2. When 孩子累计星星达到升级阈值，the system shall 展示升级动画，并推送微信服务通知。
3. The system shall 在孩子详情页显示当前等级、距下一级所需星星、进度条。

#### REQ-11 称号系统
> **User Story**：作为家长，我希望孩子获得特殊称号以表彰突出表现。

**验收标准：**
1. The system shall 内置 ≥ 15 个称号，通过特定成就自动解锁：

| 称号 | 解锁条件 |
|------|---------|
| 早起冠军 | 连续7天完成"早起"任务 |
| 阅读达人 | 累计完成"阅读"任务30次 |
| 运动健将 | 累计完成"运动"任务20次 |
| 整洁能手 | 连续14天完成"整理房间" |
| 满分宝贝 | 单日完成全部任务 |
| 百日坚持 | 累计打卡100天 |
| 学霸星人 | 累计完成学习任务50次 |
| ... | ... |

2. When 称号解锁，the system shall 展示庆祝弹窗并记录解锁时间。
3. The system shall 支持家长手动授予/撤销自定义称号（最多10个自定义称号）。
4. The system shall 在孩子详情页展示已获得的所有称号徽章。

---

### 2.5 排行榜

#### REQ-12 家庭排行榜
> **User Story**：作为多孩家庭家长，我希望看到孩子间的星星排名，激发良性竞争。

**验收标准：**
1. The system shall 展示本家庭所有孩子的星星排行榜。
2. The system shall 支持三种排行维度：**本周星星**、**本月星星**、**总星星**。
3. When 孩子只有1个，the system shall 隐藏排行榜，改为展示个人成就时间线。
4. The system shall 为每个排名位置展示金/银/铜奖牌图标（前三名）。

---

### 2.6 数据统计

#### REQ-13 统计报表
> **User Story**：作为家长，我希望查看孩子的习惯养成趋势，掌握长期进展。

**验收标准：**
1. The system shall 提供以下统计维度：
   - **打卡热力图**：过去90天的每日打卡数量
   - **任务完成率**：按任务分类统计本周/本月完成百分比
   - **星星趋势**：折线图展示近30天星星变化
   - **连续打卡天数**：当前连续天数及历史最高记录
2. The system shall 支持按孩子筛选统计数据。
3. When 数据不足7天，the system shall 展示引导提示而非空图表。

---

### 2.7 星星兑换（基础版）

#### REQ-14 星星兑换奖励
> **User Story**：作为家长，我希望设置星星可兑换的奖励，增加孩子的兑换动力。

**验收标准：**
1. The system shall 支持家长创建自定义奖励（名称、所需星星、图标）。
2. The system shall 内置 5 个示例奖励模板（看电影、游乐园、买玩具等）。
3. When 孩子星星充足，家长可点击"为孩子兑换"，the system shall 扣减相应星星并记录流水。
4. The system shall 展示兑换历史记录。

## 三、非功能需求

### 3.1 性能
- 首页加载时间 ≤ 2 秒（4G 网络）
- 打卡操作响应时间 ≤ 1 秒
- 数据统计图表渲染 ≤ 3 秒

### 3.2 兼容性
- 微信基础库版本 ≥ 2.16.0
- 支持 iOS 13+ 和 Android 8+

### 3.3 安全
- 数据访问隔离：每个家长只能访问自己家庭的数据（CloudBase 安全规则）
- 孩子数据不对外暴露
- 照片上传支持压缩，单张 ≤ 2MB

### 3.4 可用性
- 操作流程：核心打卡功能 **≤ 2 步**完成
- 离线友好：关键页面数据支持本地缓存
- 空状态设计：每个空列表都有引导插图和操作入口
