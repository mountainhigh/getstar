# 微信用户登录功能实现文档

## 📋 功能概述

为 GetStar 小程序实现了完整的微信用户登录功能，确保不同用户数据完全隔离，提供安全可靠的用户认证机制。

---

## 🔐 安全特性

### 1. 加密传输协议
- **HTTPS 加密传输**：所有数据传输均通过 HTTPS 协议
- **CloudBase SDK 自动加密**：使用官方 SDK，自动处理数据加密
- **微信认证机制**：利用微信小程序原生登录，确保身份真实性

### 2. 用户认证机制
- **自动身份验证**：利用 CloudBase 和微信的天然登录特性
- **OPENID 作为唯一标识**：每个用户在当前小程序中有唯一的 OPENID
- **UNIONID 跨应用支持**：支持同一用户跨应用识别（需绑定微信开放平台）
- **登录状态持久化**：登录信息存储在 globalData 中

### 3. 严格的权限控制
- **数据库级别隔离**：所有用户数据通过 `_openid` 字段自动隔离
- **CloudBase 安全规则**：配置了严格的安全规则，确保用户只能访问自己的数据
- **云函数权限验证**：所有云函数操作都基于用户 OPENID 进行权限验证

### 4. 数据完整性保障
- **OPENID 验证**：CloudBase 自动验证 OPENID 的真实性
- **时间戳校验**：所有记录包含创建时间和更新时间
- **事务一致性**：用户和家庭数据在同一个事务中创建

## 📊 数据库设计

### 用户表 (users)

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `_id` | String | 自动生成的主键 |
| `openid` | String | 用户的唯一标识（_openid 索引） |
| `unionid` | String | 可选，跨应用唯一标识 |
| `appid` | String | 小程序 AppID |
| `nickname` | String | 用户昵称 |
| `avatar` | String | 用户头像 URL |
| `status` | String | 账户状态（active/inactive） |
| `loginCount` | Number | 登录次数 |
| `createTime` | Date | 创建时间 |
| `updateTime` | Date | 更新时间 |
| `lastLoginTime` | Date | 最后登录时间 |

### 家庭表 (families)

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `_id` | String | 自动生成的主键 |
| `name` | String | 家庭名称 |
| `ownerId` | String | 家庭创建者用户 ID |
| `createTime` | Date | 创建时间 |
| `updateTime` | Date | 更新时间 |

### 其他表（已存在）

- `children` - 孩子信息（关联 familyId）
- `habits` - 习惯信息（关联 childId）
- `check_in_records` - 打卡记录（关联 childId）
- `titles` - 称号信息（关联 childId）
- `habit_templates` - 习惯模板（公开数据）

## 🔒 安全规则配置

### users 集合
```json
{
  "read": "auth != null && auth.openid != null && doc._openid == auth.openid",
  "write": "auth != null && auth.openid != null && doc._openid == auth.openid"
}
```
**说明**：用户只能读写自己的数据

### families 集合
```json
{
  "read": "auth != null && auth.openid != null && doc._openid == auth.openid",
  "write": "auth != null && auth.openid != null && doc._openid == auth.openid"
}
```
**说明**：家庭数据通过 _openid 隔离，确保用户只能访问自己的家庭

### children、habits、check_in_records、titles 集合
```json
{
  "read": "auth != null && auth.openid != null",
  "write": "auth != null && auth.openid != null && doc._openid == auth.openid"
}
```
**说明**：读取需要登录（通过 familyId 隔离），写入只能创建自己的数据

### habit_templates 集合
```json
{
  "read": "true",
  "write": "false"
}
```
**说明**：所有人可读，不可写（公开模板数据）

## 🚀 登录流程

### 1. 用户打开小程序
```
用户打开小程序
    ↓
app.js onLaunch 执行
    ↓
调用 checkLoginAndInit()
```

### 2. 登录初始化
```
获取 OPENID
    ↓
调用 userLogin 云函数
    ↓
检查用户是否存在
    ↓
新用户 → 创建用户记录
老用户 → 更新登录信息
    ↓
检查家庭是否存在
    ↓
无家庭 → 自动创建家庭
有家庭 → 加载家庭信息
    ↓
加载第一个孩子
    ↓
登录完成
```

### 3. 数据访问流程
```
用户操作页面
    ↓
检查登录状态 (getUserInfo)
    ↓
验证通过 → 执行操作
验证失败 → 提示登录
    ↓
操作数据（自动带 _openid）
    ↓
CloudBase 验证权限
    ↓
返回结果
```

## 📝 注意事项

1. **小程序 AppID**
   - 需要在 `project.config.json` 中配置正确的小程序 AppID
   - 否则无法正常获取 OPENID

2. **CloudBase 环境**
   - 确保 CloudBase 环境已正确配置
   - 环境状态正常

3. **UNIONID**
   - UNIONID 仅在绑定微信开放平台后可用
   - 如果未绑定，UNIONID 为 null
   - 不影响正常使用，OPENID 足够

4. **数据安全**
   - 严禁绕过安全规则直接操作数据库
   - 所有数据访问必须经过云函数或客户端 SDK
   - 不要在前端暴露敏感信息
