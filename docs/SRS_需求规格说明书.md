## 1. 引言

### 1.1 目的
本文档描述"儿童行为激励管理微信小程序"的功能需求和非功能需求，供开发、测试、产品等团队使用。该产品面向家长群体，通过游戏化星星机制帮助孩子养成良好习惯。

### 1.2 范围

**包含范围**：
- 家长端完整功能：星星规则设置、打卡审核、数据统计、星星展示、称号系统、排行榜
- 多孩家庭管理功能
- 预设习惯模板快速打卡
- 游戏化激励系统（星星、等级、称号、排行榜）
- 礼物兑换功能（金币系统、礼物管理、兑换记录）

**不包含范围**：
- 儿童端独立应用（由家长统一操作）
- 社交分享功能（本版本）
- 付费功能（本版本免费使用）

### 1.3 术语与缩写

| 术语 | 定义 |
|------|------|
| 打卡 | 家长记录孩子完成某项习惯的行为 |
| 星星 | 孩子完成习惯获得的奖励分值 |
| 金币 | 等值于星星的虚拟货币，用于兑换礼物（1星星 = 1金币） |
| 称号 | 根据星星和等级解锁的荣誉头衔 |
| 等级 | 星星累积达到的层级（如Lv.1 - Lv.20） |
| 排行榜 | 孩子星星在家庭成员中的排名 |
| 多孩家庭 | 一个家庭账户管理多个孩子 |
| 礼物 | 家长预设的奖励物品，用金币兑换 |
| 兑换 | 使用金币兑换礼物的行为 |
| 兑换记录 | 礼物兑换的历史记录 |

## 2. 整体描述

### 2.1 产品愿景
通过星星、称号、等级系统，让家长轻松管理孩子的习惯养成，让孩子在正向激励中健康成长，解决家长"催促难、坚持难"的痛点。同时通过微信账号登录和数据隔离机制，保障多用户数据安全和隐私。

### 2.2 用户群体

| 角色 | 描述 | 使用频率 |
|------|------|---------|
| 家长 | 使用微信账号登录，管理孩子的习惯、设置星星规则、审核打卡 | 高（每天多次） |

### 2.3 假设与依赖

**假设**：
- 家长已安装微信
- 家长有一定使用小程序的熟悉度
- 家长愿意花时间记录孩子行为
- 用户信任微信账号体系

**依赖**：
- CloudBase 云开发环境
- 微信小程序平台
- 微信开放平台认证服务

## 3. 功能需求

### 3.1 用户管理模块

#### FR-000：用户微信登录
- **描述**：用户使用微信账号登录，实现不同用户数据完全隔离
- **优先级**：Must Have
- **验收标准**：
  - [ ] 用户首次打开小程序自动完成微信登录
  - [ ] 通过 OPENID 实现用户身份唯一识别
  - [ ] 新用户自动创建用户记录
  - [ ] 新用户自动创建家庭记录
  - [ ] 老用户自动加载已有数据
  - [ ] 登录状态自动维护，无需手动登录
  - [ ] 实现严格的数据隔离，用户只能访问自己的数据
  - [ ] 采用 HTTPS 加密传输
  - [ ] 数据库配置严格的安全规则
  - [ ] 用户 OPENID 无法伪造和篡改

#### FR-001：家庭账户创建
- **描述**：首次使用时创建家庭账户，支持添加多个孩子
- **优先级**：Must Have
- **验收标准**：
  - [ ] 用户登录后自动创建家庭（如果不存在）
  - [ ] 支持添加1-5个孩子
  - [ ] 支持编辑孩子信息（姓名、头像、昵称）
  - [ ] 支持删除孩子（需二次确认）
  - [ ] 孩子信息关联家庭ID，确保数据隔离

#### FR-002：多孩家庭切换
- **描述**：家长可以在多个孩子之间快速切换进行操作
- **优先级**：Must Have
- **验收标准**：
  - [ ] 首页顶部显示当前选中孩子
  - [ ] 点击可切换到其他孩子
  - [ ] 切换后数据实时更新
  - [ ] 单孩家庭不显示切换功能
  - [ ] 只能切换同一家庭下的孩子

### 3.2 习惯管理模块

#### FR-003：习惯模板选择
- **描述**：提供预设习惯模板，家长可快速选择无需手动创建
- **优先级**：Must Have
- **验收标准**：
  - [ ] 提供20+预设习惯模板（阅读、运动、整理、早睡早起等）
  - [ ] 模板包含：名称、星星奖励、建议频率
  - [ ] 点击模板即可添加到当前孩子
  - [ ] 添加后可自定义修改（星星、频率）
  - [ ] 支持按类别筛选模板

#### FR-004：自定义习惯创建
- **描述**：家长可自定义创建个性化习惯
- **优先级**：Should Have
- **验收标准**：
  - [ ] 支持输入习惯名称
  - [ ] 支持设置星星奖励（1-100分）
  - [ ] 支持设置频率（每天/每周/自定义）
  - [ ] 支持选择图标和颜色
  - [ ] 支持设置启用/禁用状态

#### FR-005：习惯列表管理
- **描述**：展示当前孩子的所有习惯，支持管理操作
- **优先级**：Must Have
- **验收标准**：
  - [ ] 卡片式展示习惯列表
  - [ ] 显示习惯名称、星星、频率、今日是否完成
  - [ ] 支持编辑习惯详情
  - [ ] 支持删除习惯（需确认）
  - [ ] 支持拖拽排序

### 3.3 打卡模块

#### FR-006：快速打卡
- **描述**：家长可快速完成孩子习惯打卡，2-3步完成核心操作
- **优先级**：Must Have
- **验收标准**：
  - [ ] 首页展示今日待打卡习惯
  - [ ] 点击习惯卡片进入打卡界面
  - [ ] 支持一键打卡（无需输入文字）
  - [ ] 支持添加备注（可选）
  - [ ] 支持拍照上传证据（最多9张照片）
  - [ ] 支持修改星星（默认使用习惯设置值，可上下调整）
  - [ ] 打卡后立即反馈动画和星星增加提示

#### FR-007：打卡历史查看
- **描述**：查看历史打卡记录
- **优先级**：Should Have
- **验收标准**：
  - [ ] 支持按日期查询历史
  - [ ] 显示打卡详情（时间、星星、备注）
  - [ ] 支持日历视图展示打卡日历
  - [ ] 未打卡日期标记

#### FR-008：撤销打卡
- **描述**：支持撤销误打卡记录
- **优先级**：Should Have
- **验收标准**：
  - [ ] 支持撤销当天的打卡（24小时内）
  - [ ] 撤销后星星自动扣减
  - [ ] 需要二次确认
  - [ ] 超过24小时不可撤销

### 3.4 星星系统模块

#### FR-009：星星展示
- **描述**：实时展示孩子当前星星和等级
- **优先级**：Must Have
- **验收标准**：
  - [ ] 首页显著位置显示总星星
  - [ ] 显示当前等级（Lv.1 - Lv.20）
  - [ ] 显示下一级所需星星
  - [ ] 星星变化动画效果
  - [ ] 本周/本月星星统计

#### FR-010：等级系统
- **描述**：根据星星自动升级，每级有不同的称号
- **优先级**：Must Have
- **验收标准**：
  - [ ] 定义20个等级，每个等级有专属称号
  - [ ] 称号命名主题：成长之路（如：萌芽-幼苗-树苗-小树-大树-森林之王等）
  - [ ] 升级时触发庆祝动画
  - [ ] 升级历史记录

#### FR-011：称号系统
- **描述**：除了等级称号外，根据完成情况解锁特殊称号
- **优先级**：Could Have
- **验收标准**：
  - [ ] 连续打卡7天解锁"坚持不懈"称号
  - [ ] 单日星星超过100分解锁"超级新星"称号
  - [ ] 累计星星达到5000分解锁"星星大师"称号
  - [ ] 可佩戴称号展示
  - [ ] 称号图鉴收集

#### FR-012：星星排行榜
- **描述**：家庭成员星星排名，激发竞争意识
- **优先级**：Should Have
- **验收标准**：
  - [ ] 显示本周星星排名
  - [ ] 显示总星星排名
  - [ ] 前三名特殊标识（金/银/铜牌）
  - [ ] 支持切换时间范围（本周/本月/总榜）

### 3.5 统计分析模块

#### FR-013：打卡统计
- **描述**：查看孩子的打卡数据统计
- **优先级**：Should Have
- **验收标准**：
  - [ ] 今日打卡数、连续打卡天数
  - [ ] 本周/本月完成率
  - [ ] 各习惯完成情况图表
  - [ ] 打卡热力图（日历形式）

#### FR-014：星星趋势
- **描述**：查看星星增长趋势
- **优先级**：Could Have
- **验收标准**：
  - [ ] 折线图展示星星趋势
  - [ ] 支持按天/周/月查看
  - [ ] 标记重要里程碑（升级、称号解锁）

### 3.6 数据管理模块

#### FR-015：数据导出
- **描述**：支持导出孩子数据用于备份或分享
- **优先级**：Could Have
- **验收标准**：
  - [ ] 支持导出Excel格式
  - [ ] 包含打卡记录、星星变化、等级历史
  - [ ] 导出文件命名格式：孩子姓名_日期.xlsx

#### FR-016：数据同步
- **描述**：多设备数据同步
- **优先级**：Should Have
- **验收标准**：
  - [ ] 支持CloudBase云存储
  - [ ] 数据实时同步
  - [ ] 离线时本地缓存，联网后自动同步
  - [ ] 多个家长账号可共享数据（需家庭成员关联）

### 3.7 礼物兑换模块

#### FR-017：金币系统
- **描述**：星星等值转换为金币，用于兑换礼物（1星星 = 1金币）
- **优先级**：Must Have
- **验收标准**：
  - [ ] 金币与星星等值（1星星 = 1金币）
  - [ ] 金币是星星的等值副本，用于礼物兑换
  - [ ] 打卡时同时获得星星和金币（同步更新）
  - [ ] 首页显示金币余额
  - [ ] 金币兑换不影响星星和称号（金币独立扣减）
  - [ ] 金币变化有动画效果

#### FR-018：礼物管理
- **描述**：家长可以预设礼物，设置礼物名称、图标、所需金币
- **优先级**：Must Have
- **验收标准**：
  - [ ] 支持添加礼物（名称、图标、金币数量）
  - [ ] 支持编辑礼物信息
  - [ ] 支持删除礼物（需二次确认）
  - [ ] 图标从预设图标库选择
  - [ ] 金币数量可自定义设置

#### FR-019：礼物兑换
- **描述**：孩子可以用金币兑换预设的礼物
- **优先级**：Must Have
- **验收标准**：
  - [ ] 显示可兑换礼物列表
  - [ ] 点击兑换按钮弹出确认对话框
  - [ ] 确认后扣除金币
  - [ ] 金币不足时提示并禁用兑换按钮
  - [ ] 兑换成功后有动画效果
  - [ ] 兑换记录自动生成

#### FR-020：兑换记录查看
- **描述**：家长可以查看礼物兑换的历史记录
- **优先级**：Must Have
- **验收标准**：
  - [ ] 显示兑换记录列表（时间、礼物、金币）
  - [ ] 支持时间筛选（全部/本周/本月）
  - [ ] 显示兑换状态（进行中/已完成）
  - [ ] 支持标记兑换为"已完成"
  - [ ] 支持删除兑换记录
  - [ ] 礼物兑换线下完成，线上记录状态

#### FR-021：礼物库存管理
- **描述**：家长可以为每个孩子单独管理礼物库存
- **优先级**：Should Have
- **验收标准**：
  - [ ] 每个孩子的金币独立计算
  - [ ] 每个孩子的兑换记录独立显示
  - [ ] 礼物可以设置为"通用"或"专属"
  - [ ] 专属礼物只有特定孩子可见

## 4. 非功能需求

### 4.1 性能需求
- 页面加载时间：首屏 < 2s（4G网络）
- 打卡操作响应时间：< 500ms
- 数据列表渲染：100条记录 < 1s

### 4.2 可用性需求
- 核心功能（打卡）3步内完成
- 操作错误提示清晰明确
- 适配不同屏幕尺寸（iPhone SE到iPhone 14 Pro Max）
- 无障碍访问（字体大小可调）

### 4.3 兼容性需求
- 微信版本：8.0.0+
- 基础库版本：3.0.0+
- 操作系统：iOS 14+、Android 10+

### 4.4 可靠性需求
- 离线时支持本地操作，联网后自动同步
- 数据本地备份，防止数据丢失
- 异常情况有友好提示

### 4.5 安全需求
- 数据通过CloudBase安全存储
- 家庭成员间数据隔离
- 支持数据导出，用户拥有数据所有权

## 5. 约束条件

- **技术栈**：微信小程序 + CloudBase云开发
- **平台限制**：仅支持微信小程序
- **用户系统**：无需登录，基于微信OpenID
- **时间限制**：无明确上线时间限制
