## 模板4：非功能需求清单 {#nfr}

```markdown
# 非功能需求清单

## 性能需求
| ID | 需求描述 | 指标 | 测试方法 | 优先级 |
|----|------|------|------|------|
| NFR-P01 | 首页加载时间 | < 3s（4G网络） | Lighthouse | Must |
| NFR-P02 | API 响应时间 | P99 < 500ms | 压测 | Must |
| NFR-P03 | 并发用户支持 | 1000 并发 | JMeter | Should |
| NFR-P04 | 数据库查询时间 | < 100ms | APM | Should |

## 安全需求
| ID | 需求描述 | 指标/标准 | 优先级 |
|----|------|------|------|
| NFR-S01 | 用户密码存储 | bcrypt 加密，cost factor ≥ 10 | Must |
| NFR-S02 | 传输加密 | TLS 1.2+ | Must |
| NFR-S03 | SQL 注入防护 | 参数化查询 | Must |
| NFR-S04 | XSS 防护 | CSP 策略 + 输出编码 | Must |
| NFR-S05 | CSRF 防护 | Token 机制 | Must |
| NFR-S06 | 权限控制 | RBAC，最小权限原则 | Must |

## 可用性需求
| ID | 需求描述 | 指标 | 优先级 |
|----|------|------|------|
| NFR-A01 | 系统可用性 | 99.9%（月均） | Must |
| NFR-A02 | 故障恢复时间 | RTO < 4h | Should |
| NFR-A03 | 数据恢复点 | RPO < 1h | Should |
| NFR-A04 | 降级策略 | 核心功能可用 | Must |

## 可扩展性需求
| ID | 需求描述 | 指标 | 优先级 |
|----|------|------|------|
| NFR-SC01 | 水平扩展 | 支持无状态横向扩展 | Should |
| NFR-SC02 | 数据增长 | 支持 [N] 年数据积累 | Should |

## 可维护性需求
| ID | 需求描述 | 要求 | 优先级 |
|----|------|------|------|
| NFR-M01 | 日志完整性 | 关键操作全链路日志 | Must |
| NFR-M02 | 部署时间 | 全量部署 < 30min | Should |
| NFR-M03 | 代码覆盖率 | 单测覆盖率 > 70% | Should |
```

## 模板6：功能优先级矩阵 {#priority-matrix}

```markdown
# 功能优先级矩阵

## 评分维度说明
- **业务价值**（1-5）：对核心业务目标的贡献程度
- **用户价值**（1-5）：对用户体验的改善程度  
- **实现成本**（1-5）：开发复杂度，1=简单，5=极复杂
- **风险**（1-5）：技术/业务风险，1=低风险，5=高风险

**优先级得分** = (业务价值 × 0.4 + 用户价值 × 0.3) / (实现成本 × 0.2 + 风险 × 0.1)

## 优先级矩阵

| ID | 功能名称 | 业务价值 | 用户价值 | 实现成本 | 风险 | 得分 | 优先级 | Sprint |
|----|------|------|------|------|------|------|------|------|
| FR-001 | [功能1] | 5 | 5 | 2 | 1 | 7.67 | P0 | Sprint 1 |
| FR-002 | [功能2] | 4 | 3 | 3 | 2 | 4.20 | P1 | Sprint 2 |
| FR-003 | [功能3] | 3 | 4 | 4 | 3 | 3.50 | P2 | Sprint 3 |

## 优先级说明
- **P0 (Must Have)**：得分 > 6，首期必须上线
- **P1 (Should Have)**：得分 4-6，首期尽量上线
- **P2 (Could Have)**：得分 2-4，时间允许则上线
- **P3 (Won't Have)**：得分 < 2，本期不做
```
