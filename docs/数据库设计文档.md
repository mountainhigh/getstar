## 2. 数据库集合设计

### 2.1 families（家庭表）

**集合说明**: 存储家庭基本信息

**字段说明**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| _id | String | 是 | 主键（自动生成） |
| familyId | String | 是 | 家庭ID（唯一） |
| openId | String | 是 | 家长OpenID |
| createdAt | Number | 是 | 创建时间（时间戳） |
| updatedAt | Number | 是 | 更新时间（时间戳） |

**索引**:
- `familyId`: 唯一索引
- `openId`: 普通索引

**示例数据**:
```json
{
  "_id": "fam_abc123",
  "familyId": "family_001",
  "openId": "oxxxx",
  "createdAt": 1677696000000,
  "updatedAt": 1677696000000
}
```

### 2.4 check_ins（打卡记录表）

**集合说明**: 存储打卡记录

**字段说明**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| _id | String | 是 | 主键（自动生成） |
| _openid | String | 是 | 系统自动注入，用户的OpenID（**不可手动设置**） |
| checkInId | String | 是 | 打卡ID（唯一） |
| childId | String | 是 | 孩子ID |
| habitId | String | 是 | 习惯ID |
| habitName | String | 是 | 习惯名称 |
| habitPoints | Number | 是 | 习惯设置的星星 |
| points | Number | 是 | 实际获得的星星（可修改） |
| note | String | 否 | 备注 |
| photos | Array | 否 | 照片URL列表（最多9张） |
| timestamp | Number | 是 | 打卡时间（时间戳） |
| date | String | 是 | 日期（YYYY-MM-DD） |
| undone | Boolean | 是 | 是否已撤销 |

- `checkInId`: 唯一索引

## 3. 数据关系图

```
families (家庭)
    ↓ 1:N
children (孩子)
    ↓ 1:N
    ├── habits (习惯)
    ├── check_ins (打卡记录)
    ├── level_records (等级记录)
    ├── user_badges (用户称号)
    ├── reward_exchanges (兑换记录)
    └── coin_records (金币记录)

children (孩子) ←→ daily_records (每日记录)

habits (习惯) ←→ check_ins (打卡记录)

badges (称号定义) ←→ user_badges (用户称号)

rewards (礼物) ←→ reward_exchanges (兑换记录)

reward_exchanges (兑换记录) ←→ coin_records (金币记录)

habit_templates (习惯模板) - 独立
```

## 9. ⚠️ 重要说明

### 9.1 `_openid` 字段使用规则

**关键规则**：
1. **`_openid` 是系统保留字段，由 CloudBase 自动注入，不可手动设置**
2. **前端（小程序端）或云函数中 `add()` 操作时，不要在 `data` 中包含 `_openid` 字段**
3. 系统会自动将当前用户的 OpenID 注入到 `_openid` 字段中
4. `_openid` 用于权限控制和安全验证

**正确做法**：
```javascript
// ❌ 错误：不要手动设置 _openid
await db.collection('children').add({
  data: {
    _openid: app.globalData.openid,  // 错误！会导致 "Invalid Key Name: _openid" 错误
    familyId: familyId,
    name: '小明'
  }
});

// ✅ 正确：系统自动注入 _openid
await db.collection('children').add({
  data: {
    familyId: familyId,
    name: '小明'
  }
});
```

**查询时使用 `_openid`**：
```javascript
// ✅ 正确：查询时可以使用 _openid 作为过滤条件
const res = await db.collection('children').where({
  _openid: app.globalData.openid  // 用于权限验证
}).get();
```

**云函数中使用 `wxContext.OPENID`**：
```javascript
// ✅ 正确：云函数中通过 wxContext.OPENID 获取用户标识
const wxContext = cloud.getWXContext();
const { OPENID } = wxContext;

// 查询时使用 _openid 进行权限验证
const userRes = await db.collection('users').where({
  _openid: OPENID
}).get();

// 添加数据时不需要手动设置 _openid
await db.collection('children').add({
  data: {
    familyId: userRes.data[0].familyId,
    name: childName
    // _openid 会自动注入，无需手动设置
  }
});
```

### 9.2 OpenID vs `_openid`

| 字段 | 说明 | 使用场景 |
|------|------|----------|
| `_openid` | 系统自动注入，用于权限控制 | 数据库查询（where）、安全规则 |
| `openid` | 应用自定义字段，存储家长OpenID | 业务逻辑关联（如 children.familyId 关联） |
