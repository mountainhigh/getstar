# UI 模式与组件规范参考

## 一、核心 UI 组件规范

### 按钮（Button）

#### 层级体系
```
Primary    主要操作    填充色（品牌主色）    每页最多 1 个
Secondary  次要操作    描边（无填充）        配合 Primary 出现
Tertiary   第三级      纯文字（无边框）      低频操作
Danger     危险操作    红色填充             不可逆操作（删除）
Ghost      幽灵按钮    透明背景+描边        暗色背景上使用
```

#### 尺寸规范
```
xs:  高度 28px，内边距 8px 12px，字号 12px
sm:  高度 32px，内边距 8px 16px，字号 14px
md:  高度 40px，内边距 10px 20px，字号 14px （默认）
lg:  高度 48px，内边距 12px 24px，字号 16px
xl:  高度 56px，内边距 16px 32px，字号 18px
```

#### 状态
- Default → Hover（亮度+5%）→ Active（亮度-5%）→ Focus（描边）→ Disabled（40% 不透明度）→ Loading（spinner + 禁用交互）

---

### 输入框（Input）

#### 类型
- Text / Password / Number / Email / Search / Textarea / Select / DatePicker

#### 状态设计
```
Default:  灰色边框 1px
Focus:    主色边框 2px + 外发光
Error:    红色边框 + 错误图标 + 错误文案（边框下方）
Success:  绿色边框 + 对勾图标
Disabled: 背景 #F5F5F5 + 光标 not-allowed
```

#### 标签位置
```
顶部标签（推荐）:  最清晰，不遮挡内容
内嵌浮动标签:      节省空间，进入时上浮（Material Design）
左侧标签:          水平表单适用（标签右对齐）
占位文字:          简单表单，但输入后不可见（避免单独使用）
```

---

### 卡片（Card）

#### 结构组成
```
Card
├── Header（可选）：图片 / 图标 + 标题 + 操作
├── Body：核心内容
└── Footer（可选）：操作按钮 / 标签 / 元信息
```

#### 变体
- 基础卡片（白色背景 + 阴影）
- 描边卡片（边框无阴影，扁平感）
- 可点击卡片（hover 时阴影加深 + 位移）
- 媒体卡片（图片占满上半部分）
- 统计卡片（数字突出 + 趋势图）

#### 阴影规范
```
sm:  0 1px 2px rgba(0,0,0,0.05)           卡片默认
md:  0 4px 6px -1px rgba(0,0,0,0.1)       卡片 hover
lg:  0 10px 15px -3px rgba(0,0,0,0.1)     Modal / Dropdown
xl:  0 20px 25px -5px rgba(0,0,0,0.1)     全屏弹层
```

---

### 模态框（Modal / Dialog）

#### 使用场景
- 需要用户关注的重要操作
- 简单的确认/输入流程（≤3 步）
- 不适合全页面跳转的轻量操作

#### 规范
```
宽度: sm 400px / md 560px / lg 720px / xl 960px / full 100%
最大高度: 90vh，超出内容滚动
遮罩: rgba(0,0,0,0.5)，点击关闭（可配置）
动画: fade + scale（0.95 → 1.0），200ms ease-out
关闭按钮: 右上角，16px 大小
```

#### 按钮位置
- 操作按钮在底部右对齐（主操作在最右）
- 危险操作：主按钮使用 Danger 样式
- 多步骤：上一步 / 下一步

---

### 通知与反馈

#### Toast / Snackbar
```
位置: 右上角（桌面）/ 底部居中（移动端）
持续时间: 信息 2s / 成功 2s / 警告 4s / 错误 持久到关闭
最大堆叠: 3 条
宽度: 320-480px（桌面）/ 全宽 -32px（移动端）
```

#### Alert 横幅
```
Info（蓝）:    通知 / 说明
Success（绿）: 操作完成确认
Warning（黄）: 需要注意但不紧急
Error（红）:   操作失败 / 需立即处理
```

#### 空状态（Empty State）
```
结构:
  插图（可选，推荐）
  标题：简洁说明当前状态
  描述：解释为什么为空，或下一步
  操作按钮：引导用户填充内容

文案示例:
  - "还没有项目" → "创建你的第一个项目，开始协作"
  - "没有搜索结果" → "试试其他关键词，或清除筛选条件"
```

## 二、导航模式

### 顶部导航（Top Navigation）
- 适合：桌面 Web、内容横向展开
- 一级导航水平排列，≤7 个
- 下拉菜单用于二级导航
- 右侧放置用户信息/搜索/操作

### 侧边栏（Sidebar）
- 适合：后台管理、复杂层级导航
- 可折叠（节省空间）
- 图标 + 文字标签（折叠时只显示图标）
- 激活状态：填充色背景 + 主色文字

### 底部 Tab 栏（Bottom Tab Bar）
- 适合：移动端主导航
- 2-5 个 Tab，高频功能
- 图标 + 短标签（≤4字）
- 激活状态：主色 + 图标填充变化
- 高度：49px（iOS）/ 56-64px（Android/通用）

### 面包屑（Breadcrumb）
- 层级 ≥ 3 时使用
- 使用 "/" 或 ">" 分隔
- 当前页面不加链接
- 移动端可折叠中间层级

## 三、数据展示模式

### 表格（Table）
```
标准规范:
- 表头：背景色区分 + 字重 600
- 行高：40px（紧凑）/ 52px（默认）/ 64px（宽松）
- 斑马纹：奇偶行交替背景（可选，大量数据时用）
- 悬停：行 hover 背景高亮
- 固定列：左右固定，超宽内容滚动
- 排序：可排序列头显示排序图标
- 分页：每页 10/20/50 条，显示总数
```

### 列表（List）
```
图文列表: 左图（48-72px）+ 右侧文字（标题+描述）
纯文字列表: 标题行 + 可选描述/元信息
可选列表: 复选框 / 单选 / Toggle
可排序列表: 拖拽手柄（左侧）
```

### 统计卡片
```
主数字: 大字号（24-36px）+ Bold
对比数据: 小字号 + 趋势箭头（↑绿/↓红）
图表: 简约趋势线 / 小型柱状图
标签: 说明这个数字代表什么
```

## 四、表单设计模式

### 单步骤表单
```
布局: 单列（推荐）/ 双列（少用，只用于强相关字段）
字段顺序: 从一般到具体，从简单到复杂
分组: 用标题 + 间距分隔不同主题
必填标识: * 号（说明 * 为必填）
提交按钮: 页面底部，使用 Primary 样式
```

### 多步骤表单（Wizard）
```
步骤指示器: 顶部步骤条，显示当前步骤和总步数
进度: 可选进度条
导航: 上一步（Secondary）+ 下一步/提交（Primary）
步骤间数据持久化
最后一步: 显示摘要确认
```

### 表单校验最佳实践
```
时机:
  - 提交时: 所有字段统一校验（基础保底）
  - 失焦时: 离开字段时校验（推荐）
  - 输入时: 仅用于实时反馈（密码强度、字数限制）

错误显示:
  - 位置: 字段正下方
  - 样式: 红色 + 错误图标
  - 内容: 具体说明错误原因（"请输入有效的邮箱地址，如 name@example.com"）
  - 不要只说 "格式错误"
```

## 五、加载状态设计

### 加载类型选择
```
全页加载: 首屏内容加载（Skeleton Screen 优先于 Spinner）
局部加载: 数据刷新（内嵌 Spinner 或局部 Skeleton）
按钮加载: 提交操作（按钮内 Spinner + 禁用）
图片加载: 占位色块 → 渐入（blur-up 效果）
无限滚动: 底部 Loading 指示 + 加载中防止重复触发
```

### Skeleton Screen（骨架屏）规范
- 颜色：`#E0E0E0` 基础 + 动画（shimmer 效果从左到右）
- 结构：模拟真实内容的形状和比例
- 动画：1.5s 循环，ease-in-out
- 图片占位：16:9 / 1:1 矩形
- 文字占位：宽度变化（100% / 75% / 50% 交替）

## 六、微交互设计

### 常见微交互场景
| 场景 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| 按钮点击 | scale 0.97 | 100ms | ease |
| 卡片 hover | translateY(-2px) + shadow | 200ms | ease-out |
| Modal 出现 | fadeIn + scale 0.95→1 | 200ms | ease-out |
| Modal 消失 | fadeOut + scale 1→0.95 | 150ms | ease-in |
| Toast 出现 | slideIn from top | 300ms | spring |
| 展开/折叠 | height 0→auto | 200ms | ease-in-out |
| 页面切换 | fadeIn + translateX | 300ms | ease-out |
| 成功动效 | checkmark 描边动画 | 500ms | ease |

### 动效原则
- 有意义：动效传递信息，不只是装饰
- 快速：UI 动效通常 100-300ms，长于 500ms 感觉迟钝
- 一致性：相同操作使用相同动效
- 克制：避免过多动效造成认知负担
- 可关闭：尊重 `prefers-reduced-motion`

## 七、设计交付规范

### Figma 文件组织
```
封面页: 项目名称 + 版本 + 日期 + 负责人
---
01_设计系统: Colors / Typography / Icons / Components
02_线框图: 流程图 + 低保真草图
03_视觉稿: 各页面高保真设计稿
04_原型: 可交互流程
05_切图规范: 标注 + 导出说明
```

### 开发标注要点
```
间距: 使用设计系统变量（不写死像素）
颜色: 使用 Token 名称（Primary-500 而非 #1976D2）
字体: 注明字族、字号、行高、字重
图标: 提供 SVG 导出路径
交互: 说明所有状态（hover/focus/disabled）
响应式: 标注各断点下的布局变化
动效: 说明动效类型、时长、缓动函数
```

### 切图规范
```
图标: SVG（矢量），命名 icon-{name}.svg
插图: SVG 优先 / PNG @2x（需透明背景）
图片: WebP 格式，提供 1x / 2x
命名: kebab-case，语义化（btn-submit.png）
```
