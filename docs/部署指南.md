# GetStar 部署指南

**版本**: v1.1
**日期**: 2026-03-07
**适用阶段**: 正式运营

---

## 📋 部署前检查清单

### 环境准备
- [ ] Node.js v18+ 已安装
- [ ] 微信开发者工具已安装
- [ ] 微信小程序账号已注册
- [ ] CloudBase账号已开通（按量付费模式）

### 配置文件
- [ ] `project.config.json` 中的 `appid` 已配置
- [ ] `app.js` 中的 CloudBase 环境ID 已配置

## 🚀 部署步骤

### 第一步: 配置小程序AppID

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 注册/登录小程序账号
3. 获取小程序 AppID
4. 修改 `project.config.json`:

```json
{
  "appid": "wxXXXXXXXXXXXXXXXX", // 你的小程序AppID
  "projectname": "GetStar"
}
```

### 第二步: 配置CloudBase环境

1. 登录 [CloudBase控制台](https://console.cloud.tencent.com/tcb)
2. 创建新环境（建议选择按量付费）
3. 复制环境ID(如: `getstar-env-xxxxx`)
4. 修改 `app.js`:

```javascript
wx.cloud.init({
  env: 'getstar-env-xxxxx', // 你的环境ID
  traceUser: true,
});
```

### 第三步: 部署云函数

本项目包含多个云函数，需要全部部署。

1. 打开微信开发者工具
2. 导入项目(选择项目根目录)
3. 展开 `cloudfunctions` 目录
4. **批量部署**：
   - 右键 `cloudfunctions` 目录
   - 选择 "更多设置" -> "上传并部署：云端安装依赖（不上传node_modules）"
   - 或者逐个右键每个云函数文件夹进行部署

**核心云函数清单**：
- `userLogin`: 用户登录
- `checkIn`: 打卡
- `undoCheckIn`: 撤销打卡
- `getHabitTemplates`: 获取习惯模板
- `addHabitsFromTemplates`: 批量添加习惯
- `getLeaderboard`: 获取排行榜
- `checkBadgeUnlock`: 检查称号
- `getRewardList`: 获取礼物列表
- `saveReward`: 保存礼物
- `exchangeReward`: 兑换礼物
- `initDatabase`: 初始化数据库
- `initUserRewards`: 初始化用户礼物

### 第四步: 初始化数据库

本项目提供了自动化初始化脚本，无需手动创建集合。

1. 确保 `initDatabase` 云函数已部署成功
2. 在微信开发者工具中，打开 "云开发" 控制台
3. 选择 "云函数" -> "initDatabase"
4. 点击 "云端测试"（参数留空 `{}`）
5. 等待执行完成，查看日志确认初始化成功

**初始化内容包括**：
- 创建所有必要的数据库集合 (`users`, `families`, `children`, `habits`, `check_ins` 等)
- 导入系统预设的习惯模板 (`habit_templates`)
- 导入系统预设的称号数据 (`badges`)

### 第五步: 配置云存储权限

1. 在CloudBase控制台开通云存储
2. 设置权限:
   - 读权限: 所有用户
   - 写权限: 仅创建者（或根据业务需求配置安全规则）

### 第六步: 上传TabBar图标

确保 `images/tabbar/` 目录下包含以下图标文件：
- `checkin.png` / `checkin-active.png`
- `habit.png` / `habit-active.png`
- `history.png` / `history-active.png`
- `my.png` / `my-active.png`

如果缺失，请参考 `docs/back/TabBar图标下载完成报告.md` (已归档) 或自行下载。

## 🔧 常见问题

### Q1: CloudBase初始化失败

**错误信息**: `cloud.init is not a function`

**解决方案**:
1. 检查基础库版本 >= 2.2.3
2. 在 `project.config.json` 中设置:
```json
{
  "libVersion": "2.19.4"
}
```

### Q2: 数据库集合不存在

**错误信息**: `Collection not found`

**解决方案**:
1. 运行 `initDatabase` 云函数
2. 检查云开发环境ID是否配置正确

### Q3: 权限不足

**错误信息**: `Permission denied`

**解决方案**:
1. 检查数据库安全规则（本项目默认使用 `_openid` 进行隔离）
2. 确认用户已登录

## 📊 性能优化建议

1. **索引优化**: `initDatabase` 函数会自动创建部分索引，建议根据实际查询需求在控制台添加更多索引。
2. **云函数冷启动**: 对于高频调用的云函数（如 `checkIn`），可配置预置并发。

---

## 🔒 安全建议

1. **数据库权限**: 保持默认的安全规则，不要随意开启“所有用户可读写”。
2. **云函数权限**: 仅开放必要的云函数给小程序端调用。
