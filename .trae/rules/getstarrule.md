# 微信小程序项目规则（示例）
## 0. 项目介绍
- 项目名称：攒星星
- 项目描述：一个基于微信小程序的智能助手，帮助用户管理孩子的习惯和时间。
- 请用中文与我对话。

## 1. 目录与命名
- 页面目录统一放在 `pages/` 下，按功能模块再分子目录，如 `pages/user/`、`pages/order/`。

## 2. 代码风格
- 使用 ES6+ 语法，优先使用 `const` / `let`。
- 异步请求统一封装到 `utils/request.js`，禁止在页面直接调用 `wx.request`。
- 所有网络请求必须加 `loading` 提示，并在 `fail` 回调里统一 toast 报错。

## 3. 数据管理
- 全局数据统一放到 `app.js` 的 `globalData`，禁止跨页面直接操作其他页面的 `data`。
- 需要跨页面共享的状态，使用 `behaviors` 或 `store`（如 `westore`）集中管理。

## 4. 性能与体验
- 列表页必须做分页，单次加载数据不超过 20 条。
- 页面 `onLoad` 里禁止同步执行耗时运算，复杂计算放 worker 或异步任务。
